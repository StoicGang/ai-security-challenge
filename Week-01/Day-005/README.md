### The Challenge
<!-- What problem were you actually trying to solve today, in one sentence? -->
The goal was to build a simple semantic search pipeline that retrieves the most relevant paragraph for a natural-language query.

### Key Concept
<!-- The idea in plain English — no jargon, as if explaining to a junior. -->
1. Semantic search retrieves information based on meaning rather than exact keyword matches. Instead of comparing raw text, both the stored paragraphs and the user's query are converted into embeddings so they can be compared mathematically.
2. The same embedding model must be used for both the paragraphs and the query because embeddings generated by different models belong to different semantic spaces. Cosine similarity is only meaningful when both vectors are produced by the same embedding model.
3. Retrieval works by embedding the stored paragraphs once, embedding the user's query at search time, computing cosine similarity between the query embedding and every stored embedding, and selecting the paragraph with the highest similarity score.

### How I Approached It
<!-- Numbered steps, briefly. This is the only "steps" section — everything else below should be about the mechanism, not the process. -->
1. Receive the user's natural-language query.
2. Convert the query into an embedding using the SAME embedding model that was used for the paragraphs.
3. Compare the query embedding with every stored paragraph embedding using cosine similarity.
4. Record the similarity score for every paragraph.
5. Select the paragraph with the highest similarity score.
6. Return:
   - the original query,
   - the matching paragraph,
   - the similarity score.

paragraph_embeddings = model.encode(paragraphs)

### Key Learning
<!-- The one thing you understand now that you didn't before today. -->
1. Every document and every query must be embedded using the same embedding model.
2. Semantic search is not about matching words. By converting both documents and queries into embeddings, the system can compare their meanings mathematically and retrieve the most relevant information even when the wording is different.

### How is semantic search different from traditional keyword search?
Traditional keyword search looks for exact words or phrases in a document. If the query uses different wording, relevant documents may be missed. Semantic search instead converts both the stored documents and the user's query into embeddings that capture their meaning. It then uses cosine similarity to compare these embeddings and retrieves the documents whose meanings are most similar, even if they use different words.

Real-life Analogy:

Imagine a librarian helping you find a book.

With keyword search, you ask for a book containing the exact words "SQL Injection". If another book explains the same attack using different wording, the librarian may not find it.

With semantic search, the librarian understands what you're looking for. Even if a book discusses unsanitized user input instead of explicitly saying SQL injection, the librarian recognizes the concept and recommends it because the meanings are closely related.
