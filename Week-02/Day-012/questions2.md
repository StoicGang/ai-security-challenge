## Grounded Answer:

Based on the provided context, the formula for the Fast Gradient Sign Method (FGSM) is:

$$\mathbf{x_{adv} = x + \varepsilon \cdot \text{sign}( \nabla_x J(\theta, x, y) )}$$

Here is what each field/part of the formula does:

*   **$\mathbf{x}$ (The Input):** This is the clean input (such as an image, a file, or a data point) that the model has never seen tampered with. This is the starting point of the attack.
*   **$\mathbf{\nabla_x J(\theta, x, y)}$ (The Gradient):** Described as "the compass." 
    *   $J(\theta, x, y)$ is the loss function, which measures how wrong the model's prediction is.
    *   $\nabla_x$ means the gradient is computed with respect to the input $x$ (e.g., input pixels) rather than the weights. It determines which direction and magnitude each pixel should be pushed to make the model's loss as large (maximally wrong) as possible.
*   **$\mathbf{\text{sign}()}$ (The Equaliser):** This takes the raw gradient and keeps only the direction while discarding the magnitude (since some pixels normally influence loss a lot while others barely matter).
*   **$\mathbf{\varepsilon}$ (Epsilon):** Described as the "stealth dial" used to scale the direction.
*   **$\mathbf{x_{adv}}$:** The final adversarial output, which is created by taking the original input ($x$) and adding the scaled ($\varepsilon$) direction of maximum loss to it.
