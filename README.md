# AI Security Challenge — 8 Weeks

> Public learning repository. One week, one folder, seven days, real deliverables.
> No certificates. No tutorial copies. Just work, committed daily.

---

## Goal
Gain the skills and portfolio to obtain a **remote AI Security internship or junior AI Security Engineer role** within 8 weeks of consistent daily work.

---

## What Is in This Repository

Every week has its own folder. Inside every week folder:
- A **week README** — objective, weekly goal, resources, interview questions, quiz, reflection
- **7 day folders** (Day-001 through Day-056) — one per calendar day including weekends
- Inside each day folder, a **day README** — what to read, what to build, the exact deliverable, and a Key Insight paragraph written after completing

The Key Insight paragraph is the most important part. It must explain the **mechanism**, not the steps. If the mechanism cannot be explained in plain English, the day is not done.

## Rules

1. **No moving to the next day until today's deliverable exists on disk and is committed.**
2. **Every Key Insight paragraph explains the mechanism, not the steps.** If it reads like a tutorial walkthrough, rewrite it.
3. **Depth over breadth.** If a concept is not understood, re-read it rather than moving on.
4. **Saturday is buffer, not optional.** If a weekday deliverable is incomplete, Saturday exists to finish it before Week N+1 begins.
